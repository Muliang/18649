Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - Scenarios describe user interaction with the system. Sequence diagrams describe the scenario events in greater detail, including the message interaction between various system components.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - Describes the complex sensor, actuator, and environmental objects, including event-triggered and time-triggered behaviors requirements in the system.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - A table maintains traceability between sequence diagrams and the requirements behaviors for each module.
    • Requirements to Contstraints Traceability - A table maintains traceability between the requirements behaviors and the constraints given for each module.
    • Statecharts to Code Traceability - A table maintains traceability between state charts and the code for each module.
  • Implementation
    • Elevator Control Package - implementation of all controllers and translators which would be used in controllers.
  • Test
    • Unit Test Log - A complete list of the unit tests for each controller.
    • Unit Test Summary File parseable list of unit test files.
    • Integration Test Log - list of integration tests for each sequence diagram.
    • Integration Test Summary File - parseable list of integration test files.
    • Acceptance Test Log - List of the acceptance tests which provided by staff.
    • Fault Tolerance Test Log - Not implemented.
  • Log Files
    • Issue Log - A complete list of the design changes that you have made throughout the project
    • Improvements Log - Improvements that team members suggest or expect to see in the elevator project
    • Peer Review Log - Team members audited each others work; found issues and made suggestions for improvement.
  • Scheduling
    • Network Schedule - Rate Monotonic Analysis of CAN network for the elevator
  • Fault Tolerance
    • FMEA - Not implemented.
    • FTA - Not implemented.
  • Verification
    • High Level Requirements Verification Framework - Verification for high level requirements.
  • Presentations
    • For each presentation, place presentation file in the presentations folder and create links to each one.
